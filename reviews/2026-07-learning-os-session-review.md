# 2026-07 Learning OS Session Review

## 1. 本次会话解决了什么

这次讨论最初从 Lemon 的学习动力、写字、数学读题、遇难退出和情绪问题开始，逐步收敛成一套长期可维护的家庭学习系统。

最大的变化不是新增了多少方法，而是明确了不同层级的职责：

- 长期原则归 Learning OS；
- 学科训练方法归 Teaching Manual；
- Lemon 直接执行的动作归 Templates；
- A/B 听写本、任务手册、阅读本、作文本、练习册属于执行载体。

因此后续不再追求一个几十页甚至上百页的大而全手册，而是维护少量稳定标准。

## 2. Lemon 当前关键画像

### 学习行为

- 经常问“下一步做什么”，依赖父母调度；
- 学习持续性不足，容易以吃东西、上厕所、喊累等方式中断；
- 学习时听故事，导致核心注意力被占用；
- 任务稍难时容易心理退出，出现乱填和答非所问。

### 语文

- 写字意愿低；
- 书写质量差，需要坚持正确笔顺；
- 需要字词、阅读定位、简单表达与写作的渐进训练。

### 数学

- 读题不完整是明显失分来源；
- 计算存在基础错误；
- 明显不合理结果也未能主动发现；
- 应用题的信息提取和数量关系薄弱；
- 难题状态下能力表现明显下降。

### 英语

- 当前缺少稳定方法；
- 应以 PEP 四上为载体，优先建立听、读、说、写和延迟复习闭环。

### 情绪与规则

- 小预期不满足时曾出现跑离父母视线的危险行为；
- 强烈惩罚能短期提高执行，但容易让孩子形成“应付父母情绪”的策略；
- 需要稳定边界，而不是靠情绪强度推进。

## 3. 已冻结的长期原则

1. 不会可以，不认真不可以。
2. 情绪可以出现，责任不能因此取消。
3. 父母的情绪可以过去，任务不会自动消失。
4. 书写时正确笔顺是底线；清晰可识别是提交标准。
5. 学习时不听故事，故事作为独立活动存在。
6. 父母逐步从调度者退出，让 Lemon 自己看任务、启动、检查。
7. 普通问题轻量处理，反复错误才深度修复。
8. 减少无边界即时娱乐，但不剥夺运动、陪伴、休息等基本快乐。
9. 所有方法只训练“下一步能力”，不提前堆复杂技巧。
10. 新规则来自真实反复问题，而不是单次冲突。

## 4. 语文方向的关键决策

### 普通字与错误字分流

普通字只做认读、课文义、笔顺、规范书写、少量组词和 A/B 听写。

错误字、易混字、反复忘记字才进入“字字生花”深度修复。

“字字生花”的重点不是堆词，而是理解语义如何生长。例如：

> 潮水是一波一波地涌动；思想也会一波一波地出现和传播，所以叫“思潮”。

这类“为什么还能这样用”的解释，用于建立多维度语言理解。

### 阅读

当前只训练初级高价值能力：

> 题目关键词 → 回原文定位 → 找证据词句 → 组织一句完整答案。

暂不重点训练复杂中心思想、表达手法和文学赏析。

### 表达与写作

渐进顺序：

> 口头一句 → 写一句 → 三句话 → 短段落 → 再进入完整习作。

第一目标是愿意表达和说清楚，不是一次写出成人标准作文。

## 5. 数学方向的关键决策

数学核心定义为“信息处理”，而不只是计算。

训练闭环：

> 读题 → 定位 → 信息提取 → 关系 → 表示/模型 → 计算 → 检查。

错误必须分类，不再统一归因于“马虎”。

当前优先：

- 基础计算；
- 读完整题；
- 结果范围与合理性；
- 应用题问题和条件；
- 必要思考留痕；
- 遇难不立即退出。

舒尔特方格只作为辅助视觉扫描训练，不承担数学主训练职责。

## 6. 英语方向的关键决策

以人教版 PEP 四上为载体。

核心循环：

> 听 → 理解 → 跟读 → 认读 → 说 → 写 → 复习。

不以复杂语法为主。

单词学习应同时建立声音、词义、认读、拼写和简单句型使用。

费曼方法只做轻量版：让 Lemon 用简单语言“教爸爸这个词或句子”，不做长篇解释。

## 7. 已有执行载体

当前已明确或准备的载体包括：

- A/B 听写本：单双日交替，语文和英语使用；
- 任务手册：由 Lemon 自己记录和勾选，减少父母逐项调度；
- 阅读本/作文本：承载真实输出；
- 数学练习与错题记录；
- 舒尔特方格：已购买，后续单独定义使用方法。

原则：不为每个方法新建一本本子，优先复用已有载体。

默认内部完成：

- Audience 检查；
- 层级边界检查；
- 重复检查；
- 年龄适配检查；
- 执行可行性检查；
- Lemon 当前问题覆盖检查。

只把经过收敛的最终结果同步到仓库。
