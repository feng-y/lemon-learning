# Reviews

> Purpose：在阶段节点基于 Progress 证据做判断，并决定下一轮怎么调整。  
> Not For：不是每日学习流水账，不重复 Current Profile，也不把所有发生过的事情都永久保存。

## Review 解决什么

每次 Review 只回答：

1. 哪些长期问题真正改善了？
2. 哪些只是因为父母监督或情绪压力暂时变好？
3. 哪些干预有效，应该继续？
4. 哪些工作假设被证伪？
5. 哪些模板、提醒或外部控制可以退出？
6. 下一阶段最值得解决的 1–3 个问题是什么？

## 建议节点

- 暑假中期；
- 暑假结束；
- 四年级上期中；
- 四年级上期末；
- 其他发生明显阶段变化的节点。

## Review 输入

优先读取：

- `profile/current-stage.md`：阶段开始时的画像；
- `problems/README.md`：当前长期问题；
- 当前 `plans/`：本阶段目标和 Gate；
- `progress/`：内容推进和能力变化证据；
- 必要的真实作业、考试或错题样本。

```text

只写高信号事实，不写情绪评价。

## 3. Long-term Problems

- 变化：改善 / 无明显变化 / 恶化
- 证据：
- 当前判断：继续 / 调整 / 退出

...

## 4. 学科进展
- 语文：
- 数学：
- 英语：

重点写“哪些内容已经稳定、哪些内容暴露了新的基础问题”。

## 5. 有效与无效干预
- 保留：
- 调整：
- 停止：

## 6. 父母介入变化
- 哪类提醒减少：
- 哪类提醒仍很多：

## 7. 下一阶段决策
- 更新 Profile：是 / 否

```

## Review 之后

Review 不是终点，结论按职责更新：

- `profile/`：覆盖当前画像；
- `problems/`：继续、调整或退出长期问题；
- `plans/`：创建下一阶段计划；
- `manuals/` / `guides/`：只有发现可复用的方法缺口时才更新；
- `learning-os.md`：只有长期稳定原则真正变化时才更新。

核心原则：

> **用 Review 改判断，不用一次事件改系统。**
