# Active

这里是日常工作入口，只维护 **Lemon 现在的状态、当前问题、当前进度和阶段性信息**。

当前阶段：**2026 年暑假，三年级结束、即将进入四年级。**

## 先看什么

- [`CURRENT.md`](CURRENT.md)：当前阶段画像与本阶段重点。
- [`problems.md`](problems.md)：当前仍在持续解决的长期问题。
- [`progress.md`](progress.md)：当前学习进度与能力变化证据。
- [`notes/`](notes/README.md)：临时想法、一次观察、尚未确认的问题。

## 信息怎么放

默认先进入 `notes/`，不要因为一次现象直接修改正式结构。

经过一段时间观察后再判断：

```text
一次性现象 / 临时想法
→ notes/

当前状态发生变化
→ CURRENT.md

跨周、跨任务反复出现的问题
→ problems.md

形成了连续的改善或退化证据
→ progress.md
```

阶段结束时，再把稳定结论整理到 `history/`。

只有方法本身被多次验证、具有跨阶段复用价值时，才更新 `core/`。

> **Active 代表现在；Notes 允许杂乱；正式结论必须少而稳定。**
