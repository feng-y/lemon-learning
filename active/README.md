# Active

这里是 Lemon 当前阶段的**教学工作面**。

当前阶段：**2026 年暑假，三年级结束、即将进入四年级。**

Active 的主线不是“先找问题”，而是：

> **先明确这一阶段要学什么、怎样学、学到什么程度；在正常教学中发现薄弱点，再做针对修复。**

## 先看什么

1. [`CURRENT.md`](CURRENT.md)：当前阶段目标、教材版本和本阶段重点；
2. [`curriculum/`](curriculum/README.md)：三科当前教材知识结构和教学主线；
3. [`progress.md`](progress.md)：教材推进、掌握程度和能力变化；
4. [`problems.md`](problems.md)：持续阻碍正常学习的问题；
5. [`notes/`](notes/README.md)：临时想法、一次观察、尚未确认的信息。

## 当前三条主线

### 1. Curriculum — 正常教学

回答：**现在应该学什么？重点是什么？怎么判断学会？**

- 语文：人教版四上；
- 数学：北师大版四上；
- 英语：PEP 四上 + Power Up 3。

### 2. Progress — 掌握与成长

回答：**教材学到哪里？哪些已经掌握？哪些能力正在增长？**

既看内容进度，也看独立性、困难停留、完成质量、延迟保留和迁移。

### 3. Problems — 持续障碍跟进

回答：**哪些问题反复出现，并持续阻碍正常教学？**

Problems 是教学过程的反馈和修复机制，不是 Active 的第一入口。

## 信息怎么放

```text
教材目标 / 单元教学
→ curriculum/

掌握情况 / 连续变化证据
→ progress.md

一次现象 / 临时想法
→ notes/

跨时间、跨任务反复阻碍学习
→ problems.md
```

阶段结束后做 Review，稳定结论进入 `history/`；只有跨阶段仍有效的方法才更新 `core/`。

> **Active 代表当前教学；Curriculum 驱动学习；Progress 判断掌握；Problems 负责持续性障碍。**
