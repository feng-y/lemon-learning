# Active Notes

这里专门存放**阶段性、临时、尚未成熟的信息**。

例如：

- 今天突然想到的教育方法；
- 某次作业里观察到的新现象；
- 某次冲突后的初步判断；
- 一条还没有验证的假设；
- 一次网课、考试、听写后的即时记录；
- 还不知道应该放到哪个长期文档里的想法。

这些内容数量可以很多，不要求一开始就分类准确。

## 推荐存放方式

按日期建文件：

```text
active/notes/
  2026-07-20.md
  2026-07-23-math-reading.md
  2026-07-28-english-online-class.md
```

一个文件里可以混合多个观察，只要保留基本上下文。

## 最小记录模板

```text
# 2026-07-XX

## 发生了什么
只写事实。

## 我的想法
可以是不成熟的判断或假设。

## 可能关联
- Current：是否改变当前阶段判断？
- Problem：是否可能是反复问题？
- Progress：是否是一条进度证据？
- Teaching：是否暴露方法缺口？

## 暂不处理
没有足够证据时留在这里即可。
```

## 信息如何晋升

临时信息默认**不自动进入正式系统**。

只有在 Review 时才决定：

- 只是一次事件 → 留在 notes，之后可归档或删除；
- 是当前状态变化 → 更新 `active/CURRENT.md`；
- 是持续的长期问题 → 更新 `active/problems.md`；
- 是进度证据 → 提炼到 `active/progress.md`；
- 是可跨阶段复用的成熟方法 → 更新 `core/teaching.md`；
- 只是阶段历史 → 进入 `history/`。

核心原则：

> **临时信息可以大量存在，但正式结构只吸收被验证过的少量结论。**

这样可以自由记录，不需要每产生一个想法就重新设计仓库结构。
