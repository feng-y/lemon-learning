## 4. 跨学科能力成长

长期只看四个方向：

### Independence — 独立性

同类任务是否从成人带领逐步走向独立完成。

### Persistence — 困难停留

遇到困难时是否还能留在任务中、说出卡点并继续下一步。

### Quality — 完成质量

是否形成规范书写、完整作答和提交前自检。

### Retention / Transfer — 保留与迁移

隔一段时间、减少提示或换相近场景后是否仍会。

必要时使用 S0–S4 记录支架程度，但支架等级只是观察成人支持是否退出，不是对 Lemon 的能力标签。

## 5. Progress 怎么记录

优先记录：

> **教材 / 单元目标 + 当前掌握程度 + 暴露的盲点 + 下一步教学动作**

例如：

```text
数学某重点单元
- 核心概念：已理解
- 基础计算：稳定
- 变式题：仍不能主动识别模型
- 判断：不是继续刷基础题，下一步做 2–3 道有差异的变式并让 Lemon 解释关系
```

原始观察进入 `active/notes/`。

只有某个问题跨任务、跨时间反复出现并持续阻碍正常学习时，才进入 `active/problems.md`。

## 6. 每周 Review

每周只回答：

1. 本周教材真正推进和掌握了什么？
2. 哪些知识只是“做过”，但尚未真正理解或保留？
3. 哪个薄弱点值得下一周针对训练？
4. 哪项能力或支架正在明显改善 / 退出？
5. 是否有问题已经持续到需要进入 Problems 跟进？

阶段结束后，基于本文件和 `active/notes/` 完成 Review，并为下一阶段重建 Progress。
