## 2. 统一观察：四个长期维度

### Independence — 独立性

看完成任务需要多少外部支架。

- S0：成人示范或带着完成
- S1：关键提问后才能继续
- S2：依靠模板 / checklist 完成
- S3：基本独立，只需少量提醒或最后检查
- S4：独立完成、自检，并能在相近任务中迁移

目标不是所有任务立刻达到 S4，而是同类任务经过训练后支架逐步下降。

### Persistence — 困难停留

看遇到困难后是否仍留在任务中：

- 是否乱填或直接放弃；
- 是否能说出具体卡点；
- 是否能接受一个最小提示后继续；
- 难题后后续任务质量是否保持。

### Quality — 完成质量

看“完成”是否包含基本质量：

- 读题和作答完整；
- 书写可识别、笔顺和布局符合当前要求；
- 不漏题、不答非所问；
- 提交前发生基本自检。

### Retention / Transfer — 保留与迁移

看能力是否真正留下：

- 隔天 / 隔几天仍能主动提取；
- 减少提示后仍能完成；
- 换一道相近题、换一篇文章、换一个句子仍会；
- 模板撤掉后动作是否仍存在。

## 3. 当前问题的进展记录

Progress 不为每个 LP 建一套复杂指标。只记录**高信号变化**，优先写“场景 + 支架等级 + 结果”。

例如：

```text
2026-07-XX
- 数学常规两步应用题：从 S1 变为 S2，可按读题卡独立完成 4/5；新题仍漏 1 个条件。
- 数学新概念题：确认是知识缺口，成人示范后可完成同类题；暂不计为读题问题。
- 语文课内阅读：S2，可按定位卡找到证据；换新短文时仍需 S1 提问启动。
- 困难停留：一道陌生题没有乱填，能说出“我不知道先求哪个量”，S1 后继续。
- 英语核心词：当天 10/10，隔日无提示 8/10；2 个词重新进入 A/B。
```

原始观察放 `active/notes/`；这里保留趋势和有决策价值的证据。

## 4. 每周 Review

每周只回答五个问题：

1. 哪一类任务的支架明显下降了？
2. 哪个场景仍高度依赖 S0–S1，原因是知识缺口还是执行 / 状态问题？
3. Lemon 遇到困难时，停留和暴露卡点是否改善？
4. 哪项能力已经能隔时或换场景保持？
5. 下周哪一个支架可以尝试减少或退出？

不要为了完整而记录所有数据；只有能改变下一步教学判断的信息才进入 Progress。
