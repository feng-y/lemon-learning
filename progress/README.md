# Progress Tracking

> Audience：家长、家庭指导老师、AI  
> Purpose：长期跟踪 Lemon 的能力变化，用连续事实判断训练是否有效。  
> Not For：不是每日流水账，不以单次考试分数替代能力判断，也不用于给 Lemon 贴标签。

## 1. 为什么需要长期跟踪

`profile/current-stage.md` 描述“Lemon 现在在哪里”，但它是覆盖式快照；`progress/` 负责保留跨阶段的变化轨迹。

长期跟踪回答四个问题：

1. 哪些问题是真正持续存在的？
2. 哪些训练已经产生稳定改善？
3. 哪些外部控制可以开始撤掉？
4. 哪些问题需要升级训练方式，而不是继续加量？

因此关系是：

> **真实执行 → Progress 记录趋势 → Review 做判断 → 更新 Current Profile / Plan / Manual**

## 2. 当前基线

时间：2026 年暑假，三年级结束、即将进入四年级。

这一阶段不以“提前学完四年级”为目标，而以**利用复习和适量预习暴露并修复基础问题**为目标。

优先观察：

- 能否自己启动任务；
- 学习块内能否持续；
- 遇到稍难任务是否仍认真处理；
- 是否减少随意填写和明显离谱答案；
- 数学能否完整读题并检查；
- 汉字笔顺、清晰度和卷面是否更稳定；
- 语文阅读能否从题目回原文定位证据；
- 英语是否形成听、读、说、写和延迟复习闭环；
- 父母提醒和情绪施压是否逐步下降。

## 3. 跟踪维度

### A. 学习执行

- 独立启动率
- 父母提醒次数
- 单次有效学习时长
- 中断后返回能力

### B. 面对困难

- 随意填写次数
- 主动说出卡点次数
- 遇难后继续完成下一小步的比例

### C. 提交质量

- 漏题
- 笔顺错误
- 无法辨认或明显潦草
- 数学布局/数位问题
- 提交前自检是否发生

### D. 学科能力

语文：延迟听写、阅读定位、短表达。  
数学：读题、关系、计算、合理性检查。  
英语：听辨、认读、口头输出、拼写、延迟保留。

## 4. 记录粒度

### 日记录：只留事实

不做长篇评价，只记录当天明显事件，例如：

- 今天 4 个任务中 3 个独立启动；
- 数学 5 道应用题有 2 道未读完整题；
- 1 道难题出现乱填，提醒后能说出卡在数量关系；
- A/B 听写 10 个词，延迟正确 8 个。

### 周复盘：看趋势

每周只回答：

1. 本周最明显改善是什么？
2. 最稳定的重复问题是什么？
3. 父母哪类提醒最多？
4. 下周只改变哪 1–2 个训练动作？

### 月度/阶段复盘：更新判断

判断：

- 哪项能力从“依赖提醒”进入“能自检”；
- 哪个问题已连续改善，可以降低训练强度；
- 哪个问题连续存在，需要升级为专项训练；
- `profile/current-stage.md` 是否需要覆盖更新。

## 5. 进步的核心指标

不追求复杂评分系统。优先看五个长期趋势：

1. **父母介入下降**；
2. **独立执行上升**；
3. **遇难后退出下降**；
4. **自我检查上升**；
5. **延迟保留和迁移上升**。

成绩作为结果信号之一，但不能替代以上过程指标。

## 6. 长期记录建议

每个阶段保留一份简洁快照，例如：

- `2026-summer-baseline.md`
- `2026-fall-midterm-review.md`
- `2026-fall-final-review.md`

阶段之间重点比较变化，不追求天天写成长日记。
