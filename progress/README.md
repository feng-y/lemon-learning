# Progress Tracking

> Audience：家长、家庭指导老师、AI  
> Purpose：长期保留 Lemon 的学习推进与能力变化证据。  
> Not For：不是每日流水账，不以单次分数代替长期判断，也不重复 Current Profile。

## 1. Progress 在整个工程中的位置

`profile/current-stage.md` 描述“Lemon 现在在哪里”，它会覆盖更新；`progress/` 保留跨时间的变化轨迹。

这里回答两个不同的问题：

1. **Learning Progress**：这一阶段学了什么、复习确认了什么、预习暴露了什么？
2. **Capability Progress**：相比过去，Lemon 是否更能独立、稳定、有效地学习？

长期上，第二类比第一类更重要。

> **内容推进是训练载体；能力变化才是长期改进结果。**

## 3. Learning Progress：教学内容进度

按阶段记录即可，不需要天天维护完整课程表。

### 语文

跟踪：

- 三年级反复错误字词是否修复；
- A/B 听写的延迟保留；
- 四上生字和课文预习到哪里；
- 阅读定位和短表达训练进入什么程度。

### 数学

跟踪：

- 三年级基础计算与应用题哪些已确认稳定；
- 哪些计算或概念仍反复错误；
- 四上哪些新内容被用作诊断样本；
- 新内容暴露出哪些旧基础或信息处理问题。

### 英语

跟踪：

- PU2 已学内容哪些能脱离课堂独立输出；
- PU2 → PU3 过渡情况；
- PEP 四上哪些核心词句做过轻量预习；
- 哪些内容只是当时会，延迟后仍不稳定。

内容进度建议只记录“已确认 / 暴露问题 / 待继续”，不追求复杂百分比。

## 4. Capability Progress：长期能力变化

能力进度与 `problems/` 一一对应。

### A. 自主执行（对应 LP-01）

观察：

- 独立启动次数；
- 父母启动提醒次数；
- “下一步做什么”询问次数；
- 单次学习块完成率；
- 中断后返回率；
- 连续任务切换是否独立。

### B. 困难任务停留（对应 LP-02）

观察：

- 随意填写次数；
- 明显离谱答案次数；
- 主动说出卡点次数；
- 遇难后继续完成下一小步的比例；
- 难题是否导致后续简单任务质量一起下降。

### C. 数学信息处理（对应 LP-03）

观察：

- 漏读条件；
- 答非所问；
- 数量关系错误；
- 单位错误；
- 明显不合理结果未发现；
- 自检发现错误的次数。

### D. 提交质量（对应 LP-04）

观察：

- 笔顺错误；
- 无法辨认或明显潦草；
- 漏题；
- 布局 / 数位导致的错误；
- 被退回重做次数；
- 提交前是否自主检查。

### E. 语言信息提取与表达（对应 LP-05）

观察：

- 阅读能否根据题目定位段落；
- 能否指出证据词句；
- 脱离原文乱答是否减少；
- 一句 / 三句 / 短段落输出是否逐步独立；
- 写作启动需要多少提示。

### F. 延迟保留（对应 LP-06）

观察：

- 当天正确与次日 / 隔日正确的差距；
- A/B 听写延迟正确率；
- 反复错误内容数量；
- 脱离提示能否说、写、使用。

## 5. 记录节奏

### 日常：只留高信号事实

不要求每天写完整报告。只有明显事件值得记录，例如：

```text
2026-07-XX
- 4 个任务中 3 个独立启动；“下一步做什么”1 次。
- 数学 5 道应用题，1 道漏条件；自检发现 1 个单位错误。
- 遇到 1 道难题后没有乱填，能说出“我不知道先求哪个量”。
- 英语 A/B 听写 10 个词，当天 10/10，隔日抽查 8/10。
```

### 每周：看趋势，不做大总结

每周只需要回答：

1. 本周最明显的改善是什么？
2. 哪个长期问题最稳定地重复？
3. 父母哪类提醒最多？
4. 下周只准备减少哪 1–2 种父母介入？
5. 内容推进中暴露了什么新的基础断点？

### 阶段：形成正式 Review

暑假中期、暑假结束、期中、期末等节点进入 `reviews/`。

阶段 Review 才决定：

- 是否更新 Current Profile；
- 哪个长期问题继续、调整或退出；
- 下一阶段 Plan 的重点；
- 是否需要修改 Teaching Method。

---

## 6. 不追求复杂量化

本工程不建立复杂积分体系。

数字只用于帮助判断趋势，不应为了记录而增加家长负担。

优先看五个长期方向：

1. **父母介入下降**；
2. **独立执行上升**；
3. **遇难后的退出行为下降**；
4. **主动检查与暴露卡点上升**；
5. **延迟保留与迁移上升**。

成绩、课程进度和考试分数是重要结果信号，但不能替代这些底层能力变化。

---

## 7. 文件维护方式

建议以“阶段快照 + 阶段总结”为主：

```text
progress/
  README.md
  2026-summer-baseline.md
  2026-fall-baseline.md        # 需要时再创建
  ...

reviews/
  2026-summer-mid-review.md
  2026-summer-final-review.md
  2026-fall-midterm-review.md
  ...
```

`progress/` 保存事实和趋势，`reviews/` 保存判断和决策。不要把两者混成一份长流水账。
