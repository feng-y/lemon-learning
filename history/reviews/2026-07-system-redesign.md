# 2026-07 Lemon Learning System Redesign

## 背景

本阶段完成了从“学习资料仓库”向“长期学习改进工程”的结构调整。

最初的仓库按 `profile / problems / plans / manuals / guides / progress / reviews / docs` 平铺，虽然职责逐渐清晰，但一级目录过多，日常使用时难以回答“现在应该读什么、改什么”。

同时，真实使用中会持续产生大量临时想法、作业观察、冲突记录和阶段状态。如果每条信息都直接进入正式文档，系统会快速膨胀并失去可维护性。

## 核心决策

最终收敛为四层：

```text
core/       长期稳定原则与教学方法
active/     当前状态、当前长期问题、当前进度、临时 notes
templates/  Lemon 直接执行的轻量模板
history/    已结束阶段的快照与 Review
```

## 关键变化

### 1. 当前信息集中到 Active

`active/CURRENT.md` 成为唯一当前状态入口。

`active/problems.md` 只保存当前仍在专项解决的问题。

`active/progress.md` 同时维护当前阶段的内容进度和能力趋势。

### 2. 临时信息有默认入口

所有尚未成熟的信息默认进入 `active/notes/`。

临时信息允许大量存在，不要求立即归类。只有经过一段时间观察和 Review，才决定是否晋升为当前状态、长期问题、进度证据或长期教学方法。

### 3. 长期方法合并到 Core

不再把三本 Manual、多个 Guide 作为一级入口。

长期原则进入 `core/learning-os.md`；可复用教学方法集中到 `core/teaching.md`。

### 4. 历史与现在分离

Active 只代表现在；History 只保存过去。

阶段结束后，通过 Review 把结论沉淀到 History，再重建下一阶段 Active。

## 当前阶段

2026 年暑假，三年级结束、即将进入四年级。

当前仍以“三年级复习 + 四年级适量预习”作为诊断和训练载体，重点改善：自主执行、困难任务停留、数学信息处理、提交质量、语言信息提取与表达、延迟保留。

## 后续维护原则

> 大量真实信息进入 Notes，少量稳定结论进入 Active，更少的成熟方法进入 Core，阶段变化沉淀到 History。

仓库不追求文件越来越多，而追求长期判断越来越清楚、父母介入逐步减少。
