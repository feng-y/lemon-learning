# Phase Plans

> Purpose：把长期问题映射到一个明确时间阶段，决定这一阶段用什么学习任务验证和改善。  
> Not For：不保存永久原则，不替代 Teaching Manual，也不把每一天写成详细日程。

## 当前阶段

- [`2026-summer-transition.md`](2026-summer-transition.md)：2026 暑假，三年级结束到四年级开学前。

当前阶段定位：

> **复习用于确认旧基础是否真实稳定；预习用于提前暴露新阶段会放大的问题。**

阶段计划的目标不是覆盖更多教材，而是选择最有诊断价值和训练价值的任务，推动 `problems/` 中的长期问题改善。

## 每个 Phase Plan 至少回答

1. 当前时间和学段是什么？
2. 这一阶段最值得改善的 1–3 个长期问题是什么？
3. 复习、预习或学校学习分别承担什么作用？
4. 三科学习重点是什么？
5. 本阶段准备减少哪些父母介入？
6. 用什么 Gate 判断阶段目标是否达到？
7. 哪些未解决问题需要带入下一阶段？

## 阶段切换

建议按真实学习周期新增计划，例如：

```text
2026-summer-transition.md
2026-fall-semester.md
2027-winter-break.md
2027-spring-semester.md
```

上一阶段的结论来自 `reviews/`，不是简单复制旧计划。

新阶段开始时：

> Review → 更新 Current Profile → 确认 Active Problems → 创建新 Phase Plan

这样长期原则保持稳定，而阶段策略可以随 Lemon 的真实变化调整。
