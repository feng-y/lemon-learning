# Long-term Problem Engineering

> Audience：家长、家庭指导老师、AI  
> Purpose：维护那些跨周、跨月反复出现、不能靠一次练习解决的长期学习问题。  
> Not For：不记录一次错题、一次冲突、一次考试失误或临时情绪。

## 1. 这一层负责什么

长期问题工程是整个 Lemon Learning 的核心改进面。

它不按学科分类，而按“长期影响学习质量的问题”组织。一个问题只有满足以下条件，才进入这里：

- 在不同任务中反复出现；
- 持续时间超过一次事件；
- 会影响多个知识点或多次学习；
- 单纯增加题量不能稳定解决；
- 可以通过行为证据观察改善或恶化。

长期问题不是对 Lemon 的人格定义，也不是医学诊断，而是**当前需要被持续改善的工作对象**。

---

## 2. 问题生命周期

每个问题统一经过：

```text
Observed Facts
真实反复发生了什么
        ↓
Working Hypothesis
当前认为可能为什么发生
        ↓
Minimal Intervention
只改变一个最小关键动作
        ↓
Execution
在真实学习任务中执行
        ↓
Evidence
记录行为和结果趋势
        ↓
Review
判断假设与干预是否成立
        ↓
Continue / Adjust / Exit
继续、调整或退出专项管理
```

禁止因为一次冲突新增永久规则，也禁止因为一次表现变好就宣布问题解决。

## 3. 当前长期问题注册表

### LP-01 自主执行

**问题范围**：任务启动依赖、学习持续性、中断后无法自主返回。

当前事实：

- 经常询问“下一步做什么”；
- 学习过程中容易被吃东西、上厕所、疲劳、故事等打断；
- 需要父母持续调度才能完成连续任务。

当前目标：

> 从“父母实时调度”逐步变成“自己查看任务、启动、完成、勾选并进入下一项”。

当前最小干预：

- 每日任务外化到任务手册；
- 开始前处理喝水、上厕所、材料准备；
- 使用适龄学习块；
- 中断后返回原任务，不重新谈判是否继续；
- 父母被问“下一步做什么”时，先让 Lemon 查看任务手册。

主要证据：

- 独立启动次数；
- 父母提醒次数；
- 学习块完成率；
- 中断后返回率；
- 连续任务切换是否需要父母指令。

退出方向：大部分日常任务可以自主启动和切换，父母只需抽查。

---

### LP-02 困难任务中的停留能力

**问题范围**：任务稍难后心理退出、乱填、整页质量坍塌。

当前工作假设：

> 难度上升 → 预期失败或不舒服 → 不再认真处理信息 → 随意填写 → 等待父母情绪结束。

当前目标：

> 不会可以停在具体一步，但不能退出整个任务。

当前最小干预：

- 允许明确说“我不会”；
- 必须指出卡在哪一步；
- 任务缩小到当前最小步骤；
- 恢复思考后再继续；
- 随机答案不算完成。

主要证据：

- 随意填写次数；
- 明显离谱答案次数；
- 主动暴露卡点次数；
- 一道难题后后续简单题是否保持正常质量；
- 遇难后是否还能完成下一小步。

退出方向：面对适度困难能继续停留、表达卡点并完成必要步骤，不再出现系统性质量坍塌。

---

### LP-03 数学信息处理闭环

**问题范围**：读题不完整、漏条件、关系判断错误、答非所问、结果明显不合理但未发现。

当前目标：

> 问题 → 条件 → 关系 → 表示 / 列式 → 计算 → 检查

当前最小干预：

- 先说最后求什么；
- 圈问题、划条件；
- 说清数量关系后再列式；
- 必要时用线段图、表格或简图；
- 计算后做范围、大小、单位和题意检查。

主要证据：

- 非知识性错误数量；
- 漏条件次数；
- 答非所问次数；
- 单位错误；
- 明显不合理结果未发现次数；
- 自检发现率。

退出方向：常规应用题能稳定完成完整信息处理流程，专项读题卡和检查卡逐步退出。

---

### LP-04 提交质量

**问题范围**：笔顺、可识别书写、卷面、答题布局、漏题和提交前检查。

这里不把“写字、卷面、布局、漏题”拆成多个碎目标，而统一视为：

> **任务完成质量。**

当前底线：

- 汉字笔顺正确；
- 内容清晰可识别；
- 不明显漏题；
- 数学数位和布局不影响正确性；
- 提交前有基本检查动作。

主要证据：

- 被退回重做次数；
- 笔顺错误；
- 无法辨认内容；
- 漏题；
- 布局导致的错误；
- 自检是否真实发生。

退出方向：基本提交质量成为自动动作，父母不再逐项提醒。

---

### LP-05 语言信息提取与表达

**问题范围**：语文阅读答题脱离原文、不会定位证据，表达启动困难。

当前目标：

> 题目问什么 → 原文在哪里 → 哪些词句能证明 → 用完整的话回答。

表达从：

> 口头一句 → 写一句 → 三句话 → 短段落

逐步发展。

主要证据：

- 阅读定位准确率；
- 是否能指出证据；
- 脱离原文乱答是否减少；
- 能否独立完成一句 / 三句短表达；
- 写作启动所需提示是否减少。

退出方向：基础阅读定位和短表达成为常规能力，再进入更高阶理解与写作训练。

---

### LP-06 延迟保留与真实掌握

**问题范围**：当天看似会、第二天或离开提示后不会；语文和英语尤其明显。

当前目标：

> 从“当时记住”转向“隔一段时间仍能主动提取并使用”。

当前最小干预：

- A/B 听写交替；
- 先回忆，再看答案；
- 错误内容动态回流；
- 普通内容稳定后退出高频复习；
- 网课或明确教学后 30 分钟内做短确认。

主要证据：

- 当天正确率与延迟正确率差距；
- 反复错误内容数量；
- 不看提示能否主动说 / 写；
- 是否能把词或句型放回实际语境使用。

退出方向：延迟提取稳定，复习频率可以自然下降。

---

## 4. 问题与学科的关系

长期问题可以跨学科存在：

- LP-01 自主执行：影响全部学科；
- LP-02 困难停留：数学最明显，但不是数学专属；
- LP-03 信息处理：当前以数学为主训练场；
- LP-04 提交质量：影响全部书面任务；
- LP-05 语言提取表达：当前以语文为主训练场；
- LP-06 延迟保留：语文和英语最明显。

因此 Teaching Manual 提供训练方法，Problem Engineering 决定**为什么此时要调用这些方法，以及何时应该退出专项训练**。

---

## 5. Review 规则

长期问题不要求每天写报告。

建议：

- 日常只记录明显事实；
- 每周看趋势，不改变太多变量；
- 阶段 Review 判断问题是否继续、调整或退出。

一个问题要升级干预前，先确认：

1. 当前干预是否真的执行过一段时间；
2. 证据是否持续，而不是偶发；
3. 问题是知识缺口、执行问题还是当前方法本身无效；
4. 是否需要调整 Teaching Method，而不是继续加量。

长期问题工程的最终成功标准不是“管理得更细”，而是：

> **Lemon 可以在越来越少的外部控制下，稳定完成越来越复杂的学习任务。**
