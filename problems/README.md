# Long-term Problem Engineering

> Audience：家长、家庭指导老师、AI  
> Purpose：维护那些跨周、跨学期反复出现、不能靠一次练习解决的长期问题。  
> Not For：不记录一次错题、一次冲突或临时情绪；这些应进入日常执行记录或阶段复盘。

## 为什么需要这一层

Lemon 的很多学习问题不是单一知识点，而是长期能力与行为模式，例如：

- 每次都问“下一步做什么”，依赖父母实时调度；
- 学习过程中频繁吃东西、上厕所、听故事，持续性不足；
- 遇到稍难任务后心理退出，出现乱填和整体质量坍塌；
- 数学没有形成完整的读题—关系—计算—检查闭环；
- 写字、卷面、答题布局和提交质量不稳定；
- 阅读不会根据问题定位原文、寻找证据；
- 学过的语文/英语内容缺少延迟提取与稳定保留。

这些问题不能通过“再做十道题”直接解决，需要作为长期问题持续观察、干预、验证和退出。

## 一个长期问题的最小结构

每个长期问题至少回答：

1. **Observed Problem**：反复观察到什么事实？
2. **Impact**：它影响哪些学习任务？
3. **Working Hypothesis**：当前对原因的工作假设是什么？只是假设，不做人格标签或诊断。
4. **Intervention**：当前采用什么最小干预？
5. **Evidence**：用什么行为数据判断是否改善？
6. **Review Window**：多久复盘一次？
7. **Exit Condition**：什么情况下认为问题已不再需要专项管理？

## 当前长期问题主线

### P1. 任务启动依赖

目标：从“父母告诉下一步”逐步变为“自己查看任务手册并启动”。

主要证据：

- 每日询问“接下来做什么”的次数；
- 父母启动提醒次数；
- 能否独立完成连续任务切换。

### P2. 学习持续性与注意竞争

目标：在一个适龄学习块内持续工作，中断后能返回原任务。

当前规则：

- 开始前处理喝水、上厕所、材料准备；
- 学习时不听故事；
- 合理休息不等于重新谈判任务是否继续。

主要证据：学习块完成率、中断次数、中断后返回率。

### P3. 困难任务后的心理退出

目标：不会时能够停在具体卡点，而不是退出整个任务。

主要证据：

- 随意填写和明显离谱答案是否下降；
- 是否能说出“我卡在哪一步”；
- 一道难题后后续简单题质量是否仍稳定。

### P4. 信息处理闭环

目标：尤其在数学中形成稳定的：

> 问题 → 条件 → 关系 → 表示/列式 → 计算 → 检查

主要证据：非知识性错误、答非所问、漏条件、单位错误和检查发现率。

### P5. 提交质量

目标：把正确笔顺、可识别书写、卷面、答题布局和检查统一视为“完成质量”。

主要证据：需要退回重做的次数、书写可识别度、漏题率、自检率。

### P6. 语言信息提取与表达

目标：语文阅读能根据题目定位原文、找到关键词和证据，再形成完整回答；写作从说清楚一句逐步发展到三句和短段落。

主要证据：定位准确率、脱离原文乱答减少、短表达完整度。

## 生命周期

长期问题不是永久标签。

```text
发现反复问题
→ 建立工作假设
→ 设计最小干预
→ 在真实学习中执行
→ Progress 收集趋势证据
→ Review 判断有效性
→ 保留 / 调整 / 退出
```

只有跨场景、跨时间反复出现的问题才进入本层。问题稳定改善后应退出专项管理，避免把过去的问题永久固化成对 Lemon 的定义。
