# Lemon Learning

这是一个面向 Lemon 的**长期学习改进工程**。

它不按“语文 / 数学 / 英语 / 计划 / 模板”平铺文件，也不以提前学完多少教材为中心。整个仓库持续围绕一个问题运行：

> **Lemon 现在最影响长期学习的真实问题是什么，我们正在用什么方法改善，经过一段时间后是否真的变好了？**

当前时间点：**2026 年暑假，三年级结束、即将进入四年级。**

当前阶段使用“三年级复习 + 四年级适量预习”作为诊断和训练载体，但目标不是抢跑，而是提前发现并修复会在四年级被放大的学习问题。

---

## 一、这个工程如何运行

长期运行只有五个核心视角：

```text
1. Current Profile
   Lemon 现在处于什么状态？
            ↓
2. Long-term Problems
   哪些问题值得持续数周、数月解决？
            ↓
3. Learning Program
   当前阶段通过什么学习任务、教学方法和执行动作改善？
            ↓
4. Progress
   内容推进到哪里？能力和行为真正发生了什么变化？
            ↓
5. Review
   哪些判断成立、哪些方法有效、下一阶段应该改什么？
            ↓
   更新 Profile / Problems / Phase Plan
```

`learning-os.md` 位于这个循环底部，只提供长期稳定的教育原则与家庭边界，不参与日常进度管理。

---

## 二、Current Profile：Lemon 当前任务画像

入口：[`profile/current-stage.md`](profile/current-stage.md)

这一层只描述**现在**，回答：

- Lemon 当前处于哪个年龄和学习阶段；
- 当前有哪些可以反复观察到的学习行为；
- 哪些只是工作假设，仍需要验证；
- 当前最优先训练什么；
- 什么现象意味着可以降低父母介入。

Profile 是覆盖式快照，不保存历史，不把阶段性表现写成人格标签。

当前阶段重点包括：任务自主启动、学习持续性、困难任务中的停留能力、提交质量，以及三科学习闭环的建立。

---

## 三、Long-term Problems：长期问题工程

入口：[`problems/README.md`](problems/README.md)

这里维护的不是错题，而是**会跨任务、跨周甚至跨学期反复影响学习质量的问题**。

当前长期主线：

- 自主执行：任务启动、持续、中断后返回；
- 困难任务停留：不会时暴露卡点，而不是心理退出或乱填；
- 数学信息处理：读题、条件、关系、表示、计算、检查；
- 提交质量：笔顺、书写可识别、布局、漏题、自检；
- 语言理解与表达：语文阅读定位、证据意识、短表达；
- 稳定保留：语文 / 英语从短时记忆转为延迟提取。

长期问题统一按以下闭环推进：

> **事实 → 工作假设 → 最小干预 → 证据 → Review → 保留 / 调整 / 退出**

一个问题改善后应退出专项管理。这个工程的目标不是让问题列表越来越长，而是让需要父母专项管理的问题越来越少。

## 四、Learning Program：当前阶段如何改善

这一层由三部分组成，但它们服务同一个阶段目标，不是三个并列系统。

### 1. Phase Plan：这一阶段解决什么

当前计划：[`plans/2026-summer-transition.md`](plans/2026-summer-transition.md)

2026 暑假的核心策略：

> **复习旧内容确认基础是否稳定；预习少量新内容提前暴露下一阶段问题；把发现的问题重新路由到长期问题工程。**

阶段结束时，不以“四上学到第几单元”为核心结果，而看：

- 父母实时调度是否下降；
- 遇难后的乱填和质量坍塌是否下降；
- 数学读题与检查是否更稳定；
- 阅读是否开始形成定位—证据—回答；
- 延迟记忆是否提高；
- 提交前是否形成基本自检。

### 2. Teaching Methods：家长 / 老师 / AI 怎么教

学科方法：

- [`manuals/chinese/README.md`](manuals/chinese/README.md)
- [`manuals/math/README.md`](manuals/math/README.md)
- [`manuals/english/README.md`](manuals/english/README.md)

跨学科方法：

- [`guides/parent-intervention.md`](guides/parent-intervention.md)：判断不会、未启动、注意中断、心理退出或敷衍，只做最小必要介入；
- [`guides/capability-progression.md`](guides/capability-progression.md)：从外部规则逐步过渡到独立执行；
- [`guides/auxiliary-methods.md`](guides/auxiliary-methods.md)：A/B 听写、主动提取、间隔复习、轻量费曼、舒尔特等辅助方法及边界。

Teaching 层只维护可复用的方法，不保存 Lemon 的当前状态和历史进度。

### 3. Execution Templates：Lemon 现在怎么做

入口：[`templates/README.md`](templates/README.md)

模板把教学方法压缩成 Lemon 可以直接执行的短动作，例如阅读定位、数学读题、错误字修复和每日任务自检。

模板不是新教材，也不是一本长期填写的大册子。真实学习痕迹仍留在 A/B 听写本、任务手册、阅读本、作文本、错题本、学校作业和练习册中。

## 五、Progress：长期教学进度与能力变化

入口：[`progress/README.md`](progress/README.md)

当前基线：[`progress/2026-summer-baseline.md`](progress/2026-summer-baseline.md)

Progress 同时保留两种进度：

### 内容进度

回答“学到哪里、复习确认了什么、预习暴露了什么”。

例如：

- 三年级哪些基础已确认稳定；
- 四年级哪些内容已做诊断性预习；
- 哪些知识点仍反复出错。

### 能力进度

回答“相比过去，Lemon 是否更会学习了”。重点看：

- 独立启动是否增加；
- 父母提醒是否下降；
- 学习块持续性是否提高；
- 难题后是否仍停留在任务中；
- 非知识性错误是否下降；
- 自检是否增加；
- 延迟保留是否提高；
- 阅读定位和表达是否稳定；
- 提交质量是否更加一致。

**内容进度服务于能力改进，而不是反过来。**

## 六、Review：阶段总结与下一轮决策

入口：[`reviews/README.md`](reviews/README.md)

Review 不是学习流水账，而是阶段性判断。

建议节点：暑假中期、暑假结束、四上期中、四上期末等。

每次只回答：

1. 哪些长期问题真正改善了？
2. 哪些只是因为父母加强监督暂时变好？
3. 哪些干预有效，应该继续？
4. 哪些工作假设被证伪？
5. 哪些模板或外部控制可以退出？
6. 下一阶段最值得解决的 1–3 个问题是什么？

Review 的结果只在证据足够时更新 `profile/`、`problems/` 和下一阶段 `plans/`；只有发现可复用的方法缺口时，才修改 `manuals/` 或 `guides/`。

---

## 七、Learning OS：长期稳定底座

入口：[`learning-os.md`](learning-os.md)

这里维护不随一个学期轻易变化的原则，例如：

- 不会可以，不认真不可以；
- 任务可以暂停，但不能通过发火、拖延或乱写让任务消失；
- 先建立外部规则，再逐步转成自我管理；
- 学习与高刺激娱乐分离；
- 普通问题轻量处理，反复问题才进入专项修复；
- 父母的目标是逐步减少介入；
- 新增规则必须对应真实问题，并有验证方式和退出条件。

---

## 八、当前工程状态

**阶段**：2026 暑假，三升四过渡。

**当前主目标**：不是提前完成四年级，而是利用复习和预习，降低 Lemon 进入四年级后的结构性学习风险。

**当前判断优先级**：

1. 能否逐步摆脱父母实时调度；
2. 遇到困难时能否留在任务里；
3. 是否形成基本的读题、输出、检查闭环；
4. 提交质量是否稳定；
5. 三科学习内容是否从“当时会”转向延迟仍能提取和使用。

仓库的长期价值不在文件数量，而在于形成一条可持续多年的证据闭环：

> **看见真实问题 → 设计最小训练 → 在真实学习中执行 → 跟踪变化 → 定期总结 → 减少无效控制 → 进入下一阶段。**
