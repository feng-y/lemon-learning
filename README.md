# Lemon Learning

这是一个面向 Lemon 的长期家庭学习系统。对象是正在成长的人，因此这里不追求“流程越复杂越好”，而是追求：**规则稳定、方法清楚、逐步放手、能长期迭代。**

## 当前定位

Lemon 处于三升四阶段。当前优先级不是拔高，而是建立一套可以承载后续学习的基础系统：

- 能自己进入任务，而不是每一步都等父母安排；
- 遇到困难不立即心理退出、乱填或逃避；
- 书写遵循正确笔顺，提交内容清晰可识别；
- 数学能完整读题、定位信息、建立关系、计算并检查；
- 语文从字词理解逐步生长到阅读和表达；
- 英语以听、读、说、写和复习构成稳定语言闭环；
- 父母从“情绪施压者”转为规则维护者和学习教练。

## 角色边界

### Learning OS

读者：家长、家庭指导老师、AI。

负责“为什么这样做、长期遵循什么原则”。不承担每天具体任务。

### Current Profile

读者：家长、家庭指导老师、AI。

负责描述 Lemon 当前处于什么状态。它可以随着成长覆盖更新，不把临时问题升级为永久规则。

### Teaching Manuals

读者：家长、家庭指导老师、AI。

负责“这门学科怎么训练、孩子卡住时如何介入、什么算掌握”。不写成给 Lemon 直接阅读的理论书。

### Cross-subject Guides

读者：家长、家庭指导老师、AI。

负责跨学科但可重复复用的方法，例如家长介入、能力成长和辅助学习策略。避免在三本 Manual 中重复维护同一原则。

### Templates

读者：Lemon。

只保留今天怎么做，不解释教育理论。模板应当短、固定、可重复使用，并最终逐步退出。

### Execution Artifacts

例如 A/B 听写本、任务手册、阅读本、作文本、错题本和练习册。这些是实际记录载体，不属于系统规范本身。

## 核心原则

1. **不会可以，不认真不可以。**
2. **任务可以暂停，不能靠发火、拖延或乱写让任务消失。**
3. **先建立外部规则，再逐步转成自我管理。**
4. **学习时不听故事；听故事作为独立活动存在。**
5. **减少无边界、低成本、高刺激的即时娱乐，但不剥夺运动、陪伴、正常休息和基本快乐。**
6. **普通问题轻量处理，反复错误才进入深度修复。**
7. **Manual 维护标准，AI 按需扩展内容，孩子负责真实执行。**
8. **所有方法都服务于下一步能力，不提前训练远超当前阶段的复杂技巧。**
9. **长期原则、当前画像、教学方法和每日执行必须分层维护。**
10. **任何新增规则都应有明确问题、验证方式和退出条件。**

## 当前策略

先冻结 v1 的责任边界和最小方法。后续修改优先来自真实执行中的反复问题，而不是每次冲突后新增规则。

仓库长期维护的是**标准、方法和判断框架**，不是复制教材、堆积任务或持续扩张流程。
