# Lemon Learning

面向 Lemon 的**长期学习改进工程**。

目标不是堆积教材、计划或学习方法，而是持续回答三个问题：

1. Lemon 现在最需要解决什么？
2. 这一阶段怎样训练、怎样减少外部支架？
3. 一段时间后，能力是否真的稳定、保留并迁移了？

## Current

Lemon 10 岁，当前处于 **2026 年暑假：三年级结束、即将进入四年级**。

现阶段以 **三年级复习 + 四年级适量预习** 作为诊断和训练载体，不以提前学完整个四年级为目标。

当前重点关注：

- 从依赖父母实时调度，逐步走向自主启动和任务切换；
- 遇到困难时留在任务中，能够暴露真实卡点而不是乱填；
- 数学区分“知识 / 模型不会”和“会但不会调用”，建立完整的问题解决闭环；
- 语文加强阅读定位、口头组织和渐进表达；
- 书写、提交质量和自检逐步内化；
- 语文 / 英语从“当时会”走向延迟保留和相近场景迁移。

详细现状从 [`active/`](active/README.md) 开始阅读。

## How It Works

```text
观察真实任务
    ↓
形成当前判断
    ↓
教学 / 示范 / 临时支架
    ↓
减少支架，观察独立完成
    ↓
延迟验证与换场景迁移
    ↓
Progress / Review
    ↓
更新下一阶段判断
```

核心不是“用了多少方法”，而是：

> **同类任务是否需要越来越少的外部帮助，同时保持完成质量、延迟保留和迁移能力。**

## Start Here

```text
active/      当前工作面：状态、问题、进度和临时记录
core/        跨阶段复用的教育原则与教学方法
templates/   Lemon 尚未内化时使用的轻量执行支架
history/     已结束阶段的快照、Review 和关键结论
```

日常优先进入 [`active/`](active/README.md)。

需要教学方法时读 [`core/`](core/README.md)；需要 Lemon 直接执行的步骤时读 [`templates/`](templates/README.md)；需要比较过去阶段时读 [`history/`](history/README.md)。

## Information Rules

1. **临时想法、一次观察、尚未确认的问题** → 先进入 `active/notes/`；
2. **当前已经形成的判断** → 维护在 `active/`；
3. **多次真实使用后仍然有效、可跨阶段复用的方法** → 才进入 `core/`；
4. **阶段结束后的结论** → 沉淀到 `history/`，不继续堆在 Active。

> **临时信息可以很多，当前结论必须少而清楚，长期方法应该更少。**
