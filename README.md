# Lemon Learning

面向 Lemon 的长期学习改进工程。

目标不是堆积学习资料，而是持续回答三个问题：

1. Lemon 现在最需要解决什么？
2. 这一阶段怎样训练和验证？
3. 一段时间后，是否真的变好了？

当前阶段：**2026 年暑假，三年级结束、即将进入四年级。**

当前通过“三年级复习 + 四年级适量预习”暴露和修复问题，不以提前学完四年级内容为目标。

## 从这里开始

平时主要看 [`active/`](active/README.md)。

```text
lemon-learning/
├── active/      当前状态、问题、进度和临时记录
├── core/        长期稳定的原则与教学方法
├── templates/   Lemon 直接使用的轻量模板
└── history/     已结束阶段的快照与总结
```

### 我现在该看什么？

- 想知道 Lemon 当前状态：[`active/CURRENT.md`](active/CURRENT.md)
- 想知道正在长期解决什么：[`active/problems.md`](active/problems.md)
- 想知道最近学到哪里、有没有改善：[`active/progress.md`](active/progress.md)
- 有临时想法、一次观察或还没想清楚的信息：写入 [`active/notes/`](active/notes/README.md)

其他内容按需再读：

- 教学方法与家长介入：[`core/teaching.md`](core/teaching.md)
- 长期教育原则：[`core/learning-os.md`](core/learning-os.md)
- Lemon 执行模板：[`templates/`](templates/README.md)
- 过去阶段总结：[`history/`](history/README.md)

## 信息如何流动

```text
临时想法 / 观察
    ↓
active/notes/
    ↓
形成稳定判断后
    ↓
CURRENT / problems / progress
    ↓
阶段结束
    ↓
history/
```

只有经过反复验证、具有长期复用价值的方法，才进入 `core/`。

> **临时信息可以很多，正式结构必须很少。**
