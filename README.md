# Lemon Learning

面向 Lemon 的**长期学习与能力成长工程**。

核心目标不是持续“修问题”，而是：

> **以教材知识结构保证学习完整性，以学科能力体系决定学习深度；在正常教学中持续发现薄弱点和盲点，通过引导理解、针对训练、长期积累和正向反馈，让知识逐步转化为能力。**

## Current

Lemon 10 岁，当前处于 **2026 年暑假：三年级结束、即将进入四年级**。

当前学习主线：

- 语文：**人教版四年级上册**，重点建立字词、背诵积累、阅读理解、表达与写作的长期系统；
- 数学：**北师大版四年级上册**，围绕教材单元建立知识地图，以数学理解和数感为底层，以计算与几何为空间较大的核心训练面，配合模型与问题解决；
- 英语：**人教 PEP 四年级上册 + Power Up 3**，建立语音、词汇/Chunk、句型、基础语言规则、听说读写和长期积累系统。

暑假采用 **三年级复习 + 四年级适量预习**：复习确认基础，预习建立新学期知识地图并提前暴露断点，但不以抢跑教材进度为目标。

## Learning Loop

```text
教材知识结构 / 阶段目标
        ↓
引导理解与明确示范
        ↓
少量、有目的的针对训练
        ↓
确认掌握并暴露薄弱点 / 盲点
        ↓
针对性修复
        ↓
延迟验证与换场景迁移
        ↓
成功体验、反馈与鼓励
        ↓
逐步减少支架，进入下一学习目标
```

`problems` 只跟进那些在正常教学中**持续阻碍学习**的问题，不再驱动整个学习系统。

## Start Here

```text
active/      当前教学主线：教材、目标、进度、问题和临时观察
core/        跨阶段复用的教育原则与教学方法
templates/   Lemon 尚未内化时使用的轻量执行支架
history/     已结束阶段的快照、Review 和关键结论
```

日常优先进入 [`active/`](active/README.md)。

需要长期教学方法时读 [`core/`](core/README.md)；需要 Lemon 直接执行的步骤时读 [`templates/`](templates/README.md)；需要比较过去阶段时读 [`history/`](history/README.md)。

## Information Rules

1. **教材结构、当前学习目标和阶段推进** → `active/curriculum/` 与 `active/progress.md`；
2. **临时想法、一次观察、尚未确认的问题** → `active/notes/`；
3. **跨任务、跨时间持续阻碍学习的问题** → `active/problems.md`；
4. **多次真实使用后仍然有效、可跨阶段复用的方法** → `core/`；
5. **阶段结束后的结论** → `history/`。

> **教学是主线，问题是反馈；训练服务于理解与掌握，最终目标是能力增长和自主学习。**
