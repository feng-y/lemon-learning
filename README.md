# Lemon Learning

这是一个面向 Lemon 的**长期学习改进工程**。

它不是教材仓库，也不是把所有学习文件平铺在一起。核心目标是持续回答五个问题：

1. **Lemon 现在是什么状态？**
2. **当前最值得解决的长期问题是什么？**
3. **这一阶段准备通过什么学习任务去修复？**
4. **家长、老师和 AI 应该怎样教和介入？**
5. **经过一段时间，哪些能力真的发生了变化？**

因此仓库围绕一条长期闭环组织：

```text
Lemon 当前画像
    ↓
长期问题识别
    ↓
阶段目标与计划
    ↓
教学方法与执行模板
    ↓
真实学习与观察
    ↓
进度跟踪
    ↓
阶段总结 / Review
    ↓
更新画像、问题与下一阶段计划
```

对象是一个持续成长的人。这里不追求流程复杂，而追求：**问题可观察、训练可执行、变化可跟踪、方法可退出、父母逐步放手。**

---

## 1. Lemon：当前任务画像

[`profile/current-stage.md`](profile/current-stage.md)

这一层只回答：**现在的 Lemon 是什么状态？**

当前处于 **2026 年暑假，三年级结束、即将进入四年级**。

当前主要表现包括：

- 任务启动仍依赖父母，频繁询问“下一步做什么”；
- 学习持续性不足，容易被吃东西、上厕所、疲劳和故事打断；
- 遇到稍难任务可能心理退出，随后出现乱填和整体质量下降；
- 数学读题、信息处理和检查闭环不稳定；
- 写字、卷面、答题布局和提交质量需要持续塑形；
- 语文需要加强字词理解、阅读定位和基础表达；
- 英语需要建立稳定的听、读、说、写与延迟复习闭环。

Profile 是**动态快照**，随着成长覆盖更新。它不记录方法，也不把阶段性问题固化成人格标签。

## 2. Problems：长期问题工程

[`problems/README.md`](problems/README.md)

这一层回答：**哪些问题值得跨周、跨月持续解决？**

一次错题不进入这里；一次情绪冲突也不自动升级成长期问题。只有反复出现、跨任务影响学习质量的问题，才进入长期问题工程。

当前主线包括：

1. 任务启动依赖；
2. 学习持续性与注意竞争；
3. 困难任务后的心理退出；
4. 数学信息处理闭环；
5. 书写与提交质量；
6. 语言信息提取与表达。

每个长期问题都按同一闭环维护：

> 观察事实 → 工作假设 → 最小干预 → 行为证据 → 定期 Review → 保留 / 调整 / 退出

长期问题的目标不是永久管理，而是最终退出专项管理。

## 3. Phase：当前阶段学习工程

[`plans/2026-summer-transition.md`](plans/2026-summer-transition.md)

这一层回答：**当前阶段准备重点解决什么？**

2026 暑假是三升四的过渡窗口。当前策略不是简单“提前学四年级”，而是：

> **用三年级复习确认基础，用四年级适量预习暴露问题，再针对性修复会在新阶段继续放大的能力断点。**

复习和预习都是诊断与训练载体，不是最终目标。

暑假结束时优先判断：

- 是否更能独立启动任务；
- 学习块是否更稳定；
- 难题后是否仍能留在任务中；
- 数学读题和检查是否改善；
- 阅读是否开始形成定位与证据意识；
- 语文和英语延迟记忆是否提高；
- 父母实时提醒是否减少。

未来进入四年级上、寒假、四年级下时，新增或替换 Phase Plan，而不是重写整个系统。

## 4. Teaching：教学方法

这一层回答：**具体应该怎么教？**

### 学科教学

- [`manuals/chinese/README.md`](manuals/chinese/README.md)：语文。围绕字词理解、阅读定位、表达与写作逐步生长。
- [`manuals/math/README.md`](manuals/math/README.md)：数学。围绕读题 → 信息 → 关系 → 模型 → 计算 → 检查建立完整信息处理闭环。
- [`manuals/english/README.md`](manuals/english/README.md)：英语。围绕听 → 理解 → 跟读 → 认读 → 说 → 写 → 复习建立第二语言闭环。

### 跨学科教学

- [`guides/parent-intervention.md`](guides/parent-intervention.md)：孩子卡住时，区分“不会、未启动、注意中断、心理退出、敷衍”，选择最小必要介入。
- [`guides/capability-progression.md`](guides/capability-progression.md)：描述能力如何从外部规则逐步发展到独立执行。
- [`guides/auxiliary-methods.md`](guides/auxiliary-methods.md)：A/B 听写、主动提取、间隔复习、轻量费曼、深层转译、舒尔特方格等辅助方法及边界。

Teaching 层维护的是**可复用的方法**。它不描述 Lemon 当前状态，也不承担阶段进度记录。

## 6. Progress：长期教学进度与能力变化

[`progress/README.md`](progress/README.md)

基线：[`progress/2026-summer-baseline.md`](progress/2026-summer-baseline.md)

这一层回答：**相比过去，Lemon 真正发生了什么变化？**

这里不等于课程进度表，也不只记录“学到第几课”。长期跟踪两类进度：

### 学习内容进度

例如：

- 三年级哪些基础已复习确认；
- 四年级哪些内容已预习；
- 哪些知识点仍反复错误。

### 能力与行为进度

例如：

- 独立启动率；
- 父母提醒次数；
- 学习块持续性；
- 难题后的任务保持能力；
- 数学非知识性错误；
- 自检发现率；
- A/B 听写延迟保留率；
- 阅读定位能力；
- 提交质量稳定性。

真正重要的是第二类。课程学得更快，但长期问题没有改善，不视为核心进步。

## 7. Review：阶段总结与系统迭代

`reviews/`

这一层回答：**这一阶段发生了什么，为什么，下一步改什么？**

建议在阶段节点进行总结，例如：

- 暑假中期；
- 暑假结束；
- 四年级上期中；
- 四年级上期末。

Review 重点不是复述每天做了什么，而是判断：

1. 哪些长期问题明显改善？
2. 哪些干预有效？
3. 哪些问题只是暂时被父母高压压住？
4. 哪些工作假设被证伪？
5. 哪些模板可以退出？
6. 下一阶段最值得解决的 1–3 个问题是什么？

Review 的结论再反向更新：

- `profile/`：更新当前画像；
- `problems/`：关闭、调整或新增长期问题；
- `plans/`：制定下一阶段计划；
- `manuals/` / `guides/`：只有发现可复用的方法缺口时才修改。

## 8. Learning OS：稳定原则

[`learning-os.md`](learning-os.md)

Learning OS 位于整个工程底部，提供长期稳定的教育原则和家庭边界。它不是仓库导航，也不是当前任务列表。

当前核心原则包括：

1. **不会可以，不认真不可以。**
2. **任务可以暂停，但不能靠发火、拖延或乱写让任务消失。**
3. **先建立外部规则，再逐步转成自我管理。**
4. **学习时不听故事；故事作为独立活动存在。**
5. **普通问题轻量处理，反复问题才进入专项修复。**
6. **父母的目标是逐步减少介入，而不是永久成为实时调度器。**
7. **新增规则必须对应真实问题，并有验证方式和退出条件。**

# 长期运行方式

整个仓库最终围绕一个循环工作：

```text
            ┌──────────────────┐
            │ Current Profile  │
            │ Lemon 现在怎样   │
            └────────┬─────────┘
                     ↓
            ┌──────────────────┐
            │ Long-term Problems│
            │ 真正要解决什么   │
            └────────┬─────────┘
                     ↓
            ┌──────────────────┐
            │   Phase Plan     │
            │ 这一阶段做什么   │
            └────────┬─────────┘
                     ↓
        ┌────────────┴────────────┐
        ↓                         ↓
 Teaching Manuals             Templates
 家长怎么教                  Lemon 怎么做
        └────────────┬────────────┘
                     ↓
              Real Execution
                     ↓
                 Progress
              长期变化证据
                     ↓
                  Review
                     ↓
        更新 Profile / Problems / Plan
```

## 维护原则

- **Profile 是快照**：描述现在，不保存完整历史。
- **Problems 是问题主线**：只维护值得长期解决的问题。
- **Plan 是阶段性的**：随学期和假期变化。
- **Teaching 是方法库**：没有可复用的新认识，不轻易修改。
- **Templates 是执行接口**：短、固定、最终可退出。
- **Progress 是纵向事实**：持续保留，用于比较变化。
- **Review 是阶段总结**：把事实转化为下一轮判断。
- **Learning OS 是稳定底座**：只有教育原则真正变化时才修改。

这个仓库最终不是为了积累越来越多文档，而是为了让 Lemon 的学习改进形成一个长期、可观察、可验证、能逐步减少外部控制的工程闭环。
