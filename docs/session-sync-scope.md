# Session Sync Scope

> Purpose：说明本轮长期讨论中哪些内容值得进入仓库、应该落在哪一层，以及哪些内容只作为临时执行材料，不升级为长期规范。

## 1. 同步判断标准

一段讨论只有满足以下至少一项，才进入长期仓库：

1. 多次讨论后仍保持稳定，已经形成长期原则；
2. 能反复指导未来任务，而不是只服务某一天；
3. 明确属于某个长期责任面：Learning OS、Teaching Manual、Template、Profile/Progress；
4. 能通过真实执行观察效果，并允许后续迭代；
5. 不与已有文档重复，或能明显补齐已有文档缺口。

以下内容不直接升级为长期规范：

- 某一天的任务清单；
- 一次冲突后的临时惩罚；
- 未验证的新技巧；
- 某一课、某一道题的具体答案；
- 大量可由教材或 AI 按需生成的内容；
- 为了“看起来完整”而新增的复杂流程。

## 2. 本轮应该长期同步的内容

### 2.1 Learning OS：长期稳定原则

已经同步到 `learning-os.md`：

- 当前阶段是“外部规则塑形 + 基础学习动作自动化”，而不是拔高阶段；
- 不会可以，不认真和随机填写不算完成；
- 情绪可以存在，但不能取消任务和责任；
- 父母的情绪会过去，任务不会自动消失；
- 学习与故事、视频等语言刺激分离；
- 笔顺正确、提交可识别、安全边界属于长期底线；
- 父母介入必须逐步退出，目标是自我启动、自我检查和自我管理；
- 普通问题轻量处理，反复问题才进入深度修复。

### 2.2 Lemon 当前阶段画像

需要单独维护，避免长期原则和个体状态混在一起。

同步到 `profile/current-stage.md`：

- 当前已观察到的学习行为；
- 对这些行为的工作假设；
- 哪些是假设、哪些是事实；
- 当前优先能力；
- 哪些变化代表真正进步。

Profile 是动态文件，可以覆盖式维护；Learning OS 不应因为 Lemon 某一阶段变化而频繁改写。

### 2.3 三本 Teaching Manual

已同步：

- `manuals/chinese/README.md`
- `manuals/math/README.md`
- `manuals/english/README.md`

三本手册只维护“怎么训练”，不维护具体教材内容。

当前冻结的学科主线：

- 语文：字 → 词 → 句 → 篇 → 阅读 → 表达 → 写作；
- 数学：读题 → 定位 → 信息提取 → 关系 → 表示/模型 → 计算 → 检查；
- 英语：听 → 理解 → 跟读 → 认读 → 说 → 写 → 复习。

### 2.4 家长介入指南

需要独立成文，因为它跨越三科，是当前家庭执行中最容易漂移的部分。

同步到 `guides/parent-intervention.md`：

- 如何区分“不会”“没启动”“注意中断”“心理退出”“敷衍”；
- 什么时候直接教；
- 什么时候只问问题；
- 什么时候缩小任务；
- 什么时候退回重做；
- 如何避免把家长情绪本身变成孩子真正应付的对象。

### 2.5 能力成长与评估框架

需要独立维护，避免评价再次退回“分数/对错/粗心”。

同步到 `guides/capability-progression.md`：

- 从外部控制到独立执行的阶段；
- 如何观察启动、持续、卡点表达、检查和修复；
- 学科能力的最小成长路径；
- 什么情况下应该升级难度，什么情况下应该继续打基础。

### 2.6 Templates

已同步到 `templates/README.md`。

模板只负责把成人方法压缩成 Lemon 能执行的动作，不新增一本大而全的 Mission Book。

当前保留：

- 错误字学习卡；
- 阅读定位卡；
- 三句话表达卡；
- 数学读题卡；
- 数学检查卡；
- 英语单词轻量卡；
- 每日任务卡。

### 2.7 辅助方法边界

需要记录，但不应进入 Core 主流程。

同步到 `guides/auxiliary-methods.md`：

- A/B 交替听写；
- Retrieval Practice / 主动提取；
- Spaced Repetition / 延迟复习；
- 轻量费曼；
- 深层转译；
- 舒尔特方格；
- 学习块/Pomodoro。

这些方法只能解决特定问题，不能反过来成为新的流程中心。

## 3. 不建议长期同步的内容

### 3.1 Summerbook / 大型暑期手册

价值偏低。阶段计划可以放入 `plans/`，但不作为长期核心资产。

### 3.2 大量逐字逐课内容

例如四上所有生字完整字典、每课固定答案、整册单词卡。教材和 AI 可以按需提供，仓库长期维护标准而不是复制内容。

### 3.3 大量 Mission Cards

Lemon 需要少量固定模板，而不是几十页理论或重复任务卡。

### 3.4 复杂成人化阅读分析

当前不进入主线。Lemon 现阶段先稳定“题目理解 → 原文定位 → 证据 → 简短回答”。

### 3.5 系统化英语语法

当前不进入主线。四上阶段优先建立可理解输入、句型使用和真实输出。

## 5. 演进规则

仓库后续只允许三类变化：

1. **真实执行证明某个长期原则需要调整** → 修改 `learning-os.md`；
2. **某一学科出现可重复的问题或更有效方法** → 修改对应 Teaching Manual；
3. **Lemon 当前能力发生明显变化** → 更新 Profile、Progress 和 Capability Progression。

不要因为单次冲突就新增规则，也不要因为发现一个新方法就扩张主流程。
