# Capability Progression

> Audience：家长、家庭指导老师、AI  
> Purpose：定义 Lemon 当前阶段能力如何逐步成长，以及何时应该增加难度、减少外部控制。  
> Principle：看长期趋势，不用一次考试或一次冲突定义能力。

## 1. 总体成长路径

Lemon 当前不是单纯“知识不足”，而是需要同时建立任务执行、困难耐受、规范输出和学科学习能力。

总体路径：

```text
外部规则
→ 共同执行
→ 提示下自检
→ 独立执行
→ 自我调整
```

目标不是永久加强管理，而是让管理逐步退出。

## 2. Level 0：强外部依赖

典型表现：

- 每一步都问家长做什么；
- 遇到困难容易停止或随意填写；
- 需要强情绪或高频提醒才能推进；
- 缺少检查；
- 任务中断后不主动返回。

当前 Lemon 在部分场景仍处于这一层。

训练重点：

- 任务可视化；
- 固定学习块；
- 固定提交标准；
- 不会时说出卡点；
- 任务中断后返回原任务。

退出信号：

- 开始出现自主看任务和勾选；
- 能在一个学习块内完成大部分任务；
- 遇到难题不立即整页崩溃。

## 3. Level 1：规则下稳定执行

典型表现：

- 知道固定流程，但仍需要少量提醒；
- 能按模板完成读题、检查或听写；
- 不会时有时能主动说明；
- 错误后能够返回修复。

家长角色：

- 从实时指挥改为关键节点提醒；
- 只检查结果和少数高风险步骤。

退出信号：

- 提醒次数持续下降；
- 模板动作开始自动化；
- 能自己发现一部分错误。

## 4. Level 2：独立完成与自检

典型表现：

- 能自己启动日常任务；
- 能决定下一步；
- 能完成基础检查；
- 能把“不会”定位到具体环节；
- 遇到困难仍能停留在任务中。

家长角色：

- 抽查；
- 对新知识教学；
- 对反复问题做专项修复。

退出信号：

- 大多数基础任务无需监督；
- 能主动调整方法；
- 错误不再大量重复。

## 5. Level 3：初步自我管理

典型表现：

- 能规划一个短学习时段；
- 能根据结果决定是否复习；
- 知道什么时候需要求助；
- 能主动复盘“错在哪一步”。

这不是当前阶段的直接要求，但所有现有规则都应朝这个方向退出。

## 6. 跨学科基础能力

### 6.1 任务启动

成长路径：

> 父母告诉下一步 → 看任务手册后回答下一步 → 自己启动 → 自己安排短时顺序。

观察指标：

- 每日询问“做什么”的次数；
- 从提醒到真正开始所需时间。

### 6.2 持续执行

成长路径：

> 高频中断 → 完成 20 分钟左右学习块 → 中断后主动返回 → 稳定完成更长任务。

观察指标：

- 无关中断次数；
- 中断后是否返回；
- 父母拉回任务的次数。

### 6.3 面对困难

成长路径：

> 随意填写/退出 → 能停下 → 能说卡点 → 能做最小下一步 → 能尝试替代方法。

观察指标：

- 明显离谱答案数量；
- “我不会”之后是否能具体说明；
- 难题是否影响后续简单题。

### 6.4 提交质量

成长路径：

> 父母逐项纠正 → 按固定清单完成 → 提交前自检 → 质量稳定。

包括：

- 笔顺；
- 可识别性；
- 不漏题；
- 数学布局和单位；
- 基础检查。

## 7. 语文能力成长

当前路径：

```text
字词准确
→ 理解语境
→ 根据问题定位原文
→ 用完整句回答
→ 三句话表达
→ 短段落
→ 完整习作
```

当前不急于升级的信号：

- 仍频繁出现基础错字；
- 阅读答案脱离原文；
- 三句话仍需要大量代写。

可以升级的信号：

- 错误字延迟复测稳定；
- 阅读能独立定位证据；
- 三句话能围绕主题自然完成。

## 8. 数学能力成长

当前路径：

```text
看清题目
→ 知道求什么
→ 提取条件
→ 说清关系
→ 选择表示
→ 正确计算
→ 判断合理性
→ 独立检查
```

当前不急于增加难度的信号：

- 仍大量漏条件；
- 关系没说清就随机列式；
- 明显不合理结果不能发现；
- 一遇难题就整体退出。

可以增加难度的信号：

- 基础题流程稳定；
- 稍难题能停留并说出卡点；
- 检查动作开始自动化。

## 9. 英语能力成长

当前路径：

```text
听音识别
→ 看词认读
→ 理解基本意思
→ 跟读
→ 句型替换
→ 极短口语输出
→ 简短书写
→ 延迟复习稳定
```

当前不急于升级的信号：

- 只会中文到英文机械默写；
- 会看不会听；
- 会背单词但不会放进句型；
- 不愿开口。

可以升级的信号：

- 熟悉词听、读、义、用形成连接；
- 能在熟悉句型里自由替换；
- 第二天仍能主动回忆。

## 10. 难度调整原则

增加难度必须建立在基础动作稳定之上。

建议：

> 2 个有把握任务 + 1 个需要认真思考的任务。

如果难任务导致整个任务质量明显崩塌，应先降低单次负荷，而不是简单判断“抗压能力差”。

## 11. 评估周期

### 每日

只看执行：

- 是否启动；
- 是否完成；
- 是否检查；
- 是否出现随机填写。

### 每周

看趋势：

- 哪类错误重复；
- 哪些提示可以撤掉；
- 哪种模板仍有必要；
- 哪项能力可以提高一点难度。

### 每月/阶段

看系统变化：

- 父母介入是否下降；
- 独立任务比例是否提高；
- 面对困难的停留能力是否增强；
- 学科学习是否从“当场会”变成“延迟仍会”。

## 12. 规则升级条件

只有满足以下条件，才把某个问题升级为长期规则或专项流程：

1. 反复出现；
2. 明确影响学习质量；
3. 现有流程无法解决；
4. 新方法有清晰退出条件。

避免为每次错误新增一个永久规则。
